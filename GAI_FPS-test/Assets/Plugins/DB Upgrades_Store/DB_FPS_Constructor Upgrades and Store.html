Package Description and Contents

Thank you for purchasing the Dastardly Banana Unity FPS Constructor Weapons Starter Kit from ActiveDen. This starter kit extends our very successful free package with the following additional capabilities and assets:

  • A powerful, easy to use system to define and apply upgrades to existing weapons. This allows you to give your players more variation and control without requiring you to create separate weapons.
  • A flexible store system that allows your players to buy, upgrade and equip their weapons within the game. The look and feel of the store can be customized with no programming or it can be used as an example to build your own store.
  • Two new weapons. An M1911 pistol with upgrades and a new Assault Rifle.
  • An animated target prefab which gives you money when shot.
  • A weapon pickup system integrated with the store and upgrade functionality

This package includes the full Dastardly Banana Base FPS Constructor Weapons package so no additional downloads are needed. Features of the base package include.

  • A single weapon script which can be used for guns, lasers, melee weapons or launchers
  • Weapons can have both primary and secondary functions (such as a rifle with a grenade launcher on it)
  • Includes Aim-Down-Sights capability for weapons (as seen in such games as Borderlands and Modern Warfare 2)
  • Make weapon add-ons such as sights, scopes, and attached secondary weapons
  • Realistic accuracy: muzzle climb and bullet spread increase as weapons are fired
  • Smart crosshair adapts to the accuracy of the weapons
  • Player Controller can sprint, jump and crouch
  • Weapons sway realistically as the player moves
  • The weapon management script makes it easy to give the player weapons (up to 10 at once)
  • Bullet penetration
  • Custom Editors make configuration fast and easy

The base package is documented in a set of video tutorials at http://www.dastardlybanana.com/FPSConstructorWeapons.htm and written documentation will be available soon.

  • A Unity 3.x package containing a configured demo scene and all assets.
  • A Unity 3.x project containing the same materials as the package. The package is easier to import into an existing project but you must configure the Input key bindings as described below.
  • This documentation in PDF and HTML

Exploring the Demo Scene

The demo scene allows you to earn money by shooting targets. This money can be used to buy weapons and upgrades in the store. To enter or exit the store just press escape.

 Once in the store you can buy weapons, buy upgrades to the weapons and then equip the upgrades. You can also pickup weapons placed in the scene and sell weapons (at a discounted price of course!).

When you buy a weapon it is equipped in the first empty slot that can hold that type of weapon. If no allowed slots are empty the newly purchased weapon is not automatically equipped but can be selected using the store interface.

When you pickup a weapon it is equipped in the first empty slot that can hold that type of weapon but, unlike the store interface, if no allowed slots are empty then it is swapped with the weapon in the first allowed slot.

As you equip upgrades you'll see that equipping an upgrade may deactivate other incompatible upgrades.   You'll also notice that certain weapons are locked and that weapon slots are restricted to certain types of weapons.

The documentation below explains how to configure the store and the weapon upgrades. The best way to learn the package is to read the documentation and then make changes to the demo scene.

Configuring Weapon Upgrades

Overview of the Upgrade System

Every Upgrade is made of of two parts; the script “Upgrade.js” which controls the upgrade, and multiple upgrade component scripts which define what aspects of the weapon the upgrade affects. Every component script affects only one value, so most upgrades will have multiple of these component scripts to get their desired effect. 

Both the “Upgrade” script and all of the component scripts can be accessed in the assets folder under DBPro--->Upgrades--->Components.

Upgrade Components

The upgrade components are organized by type: the ‘Boolean’ folder has components which affect booleans (such as whether or not the gun is scoped, or has infinite ammo), the ‘Float” folder holds multipliers for float values, and so on.

Each upgrade component type behaves in a specific way, and has only one variable to worry about (with a couple special cases we’ll get into later). One factor which is important to keep in mind about these upgrade components is how they overlap; meaning whether or not having two of the same component active at the same time on any given gun would be a problem. The way the upgrades work is to have each component apply, and should there be occasion unapply, its single change. Having multiple of the same component (for components that can't overlap) can prevent the changes from being undone.

        To help understand how the components work (and which can overlap), here is a list of basic types and what their variables do along with a listing of the provided upgrade components:

  1. Boolean: sets the target variable to be equal to a value (true or false). Boolean components cannot overlap safely: only one upgrade at a time should change any given boolean variable.
  2. Float(Multiplier): multiplies the target variable by the Float’s Multiplier value. Multiple Float(Multiplier) components can safely overlap, so you can have multiple upgrades change the same float value at the same time without issue. It is important to remember that since it is a multiplier, this component will have no effect on variables with a value of 0.
  3. Integer: sets the target variable to be equal to a given integer value. Integer components can safely overlap, so you can have multiple upgrades change the same integer at the same time without issue.
  4. Vector3: used to modify gun positions, sets the target variable to be equal to a given Vector3. Vector3 components cannot overlap safely: only one upgrade at a time should change any given Vector3 variable.

Special Upgrade Components

There are three upgrade components, located in the ‘Other’ folder under ‘Components’, which don’t fit in with the basic types; GraphicsObject, MaterialChange, and SecondaryWeapon.

  1. GraphicsObject: GraphicsObject activates a specific graphical element when activated, which is useful for upgrades such as a scope or silencer. To make it work, just put the component onto your upgrade and drag the object you want activated into the ‘Obj” field.
  2. MaterialChange: MaterialChange basically finds all instances of a specific material (Start Mat) on a weapon and changes them to a different material (Target Mat). It’s useful for special finishes or camouflage. Since it only changes one material, you might need to add multiple instances of this component onto an upgrade if you need to change multiple materials. This component doesn’t work if the material is already being changed, so only one upgrade at a time should change the same material (for instance, all camouflage upgrades should be of the same type, so that only one can be active).
  3. SecondaryWeapon: SecondaryWeapon changes the secondary weapon of a given gun. For this component to work, the secondary gunscript must already be on the weapon (just not set as the secondary weapon). Then just put the number gunscript that you want to be set as the secondary in for ‘s’ (count down the gunscripts after the first, then put the number of the one you want in ‘s’). For example, if there are two additional gunscripts on the weapon and you want to activate the second, then ‘s’ would be ‘2’. Again, only one instance at a time of this component should be active for any given weapon.

Configuring an Upgrade

Actually making an upgrade is very simple: 

  1. Drag the Upgrade prefab (just an empty gameObject with the “Upgrade” script attached”) onto the weapon you want it to modify. Note that upgrades must be direct children of the weapon they modify.
  2. Set the name, description. If you are using the DB Pro Store prefab as well, then set the buy/sell prices, and owned and locked (owned if the upgrade should be already owned on start, locked if it shouldn’t be purchasable at first). More detail on configuring the upgrades for the store is provided later in this document.
  3. Set the “Upgrade Type”, which can be any String. Only one upgrade of any given type can be active, so if you have two upgrades which can’t both be active at the same time (we’ll get into that more later), just set their upgrade types to be the same and they will never overlap.
  4. Now comes the last, and most important, part- setting up the upgrade components. All you need to do is find the components which you want to use (they are named by the variable they change), and drag them onto your upgrade.

As an example, in the demo scene the M1911 has an upgrade attached called Hi-Power. This upgrade consists of 8 upgrade components.

  1. Damage - This float upgrade component increases the damage by a factor of 4
  2. Penetrate Val - An integer upgrade component that sets the bullet penetration value to 5
  3. Fire Rate - a float upgrade component that increases the fire rate by a factor of 2
  4. Volume - a float upgrade component that increases the gun volume by a factor of 2
  5. Pitch - a float upgrade component the lowers the pitch to 65% of the previous value.
  6. Ammo Per Clip - an integer upgrade component that sets the  ammoPerClip to 6
  7. Force - A float upgrade component that increases the bullet force by a fact or 7
  8. Reload Time - a float upgrade component that  increases the reloadTime by 50%

Also note that the Upgrade Type is set to 'override' so any other upgrade component of type 'override' will be disabled when Hi-Power is enabled.

Other examples in the demo scene show how to change the materials on a weapon, how to activate a scope and change the zoom, how to activate a secondary weapon, how to change the ammo and more.

Creating New Upgrade Components

The upgrades package comes equipped with a variety of upgrade components, but it’s possible that you might want to expand the library of components. This quick scripting guide will show you how to easily code upgrade components.

Each upgrade component must implement three functions.

  • Start() where it finds the script or object it needs to change
  • Apply() where it enacts that change
  • Remove() where it undoes the change.

Here’s an example of a basic upgrade component (BurstFire):

var gscript : gunscript;

var val : boolean;

var cache : boolean;

private var applied : boolean = false;

function Start () {

        var gscripts = this.transform.parent.GetComponents(gunscript);

        for(var q = 0; q < gscripts.length; q++){

                if(gscripts[q] != null && gscripts[q].isPrimaryWeapon){

                        gscript = gscripts[q];

                }

        }

}

function Apply () {

        cache = gscript.burstFire;

        gscript.burstFire = val;

}

function Remove () {

        gscript.burstFire = cache;

}

Basically, the primary gunscript is found in Start(), then in Apply() the value (burstFire in this case) is changed to be the value we want (val) and the original value is stored in cache. Finally, in Remove() the value is reverted to the cached value. 

All of the other standard types work in a similar way, although the value is modified slightly differently. If you’re trying to make a new component script for ‘gunscript’ or ‘aimmode’ in the weapons package, you can just copy & paste a component script of the same kind and replace the name of the variable. Otherwise, just follow the basic template shown above.exported.

The Upgrade.js script controls the individual upgrade components and shouldn't need to me modified.

//Upgrade.js

var applied : boolean = false;

var owned : boolean = false;

var locked : boolean = false;

var upgradeType : String;

var upgradeName : String;

var description : String = "Upgrade Locked";

var lockedDescription : String;

var buyPrice : float;

var sellPrice: float;

function ApplyUpgrade () {

        var upgrades= this.transform.parent.GetComponentsInChildren(Upgrade);

        for(var i = 0; i < upgrades.length; i++){

        if(upgrades[i].upgradeType == upgradeType && upgrades[i] != this)

                upgrades[i].RemoveUpgrade();

        }

        if(applied)

                return;

        this.SendMessage("Apply");

        applied = true;

        this.SendMessageUpwards("ApplyUpgrade");

        transform.parent.GetComponent(gunscript).deselectWeapon();

}

function RemoveUpgrade () {

        if(!applied)

                return;

        this.SendMessage("Remove");

        applied = false;

}

function DeleteUpgrade () {

        RemoveUpgrade();

        Destroy(this);

}

Configuring The Store

By default the store is activated using a 'Store' input in defined in the Input Manager. In the demo scene Store is mapped to the escape key. In order to access the store you need to go to Edit->Project-Settings->Input and define a new Input named Store.

When the store is first activated it displays 3 panels, the Header Panel (H), the Left Selection Panel (LS) and the Main Display Panel (MD).

The Header Panel lets the player choose Buy or Equip. If Buy is Selected the Left Selection Panel displays the Weapon Classes with the number of weapons in each class. The Left Selection Panel automatically adds a Next and Previous navigation arrow if needed. In Buy Mode the Main Display Panel shows the Weapons within a class and displays a green check mark if the weapon is owned and a locked icon if the weapon is locked.

If the player clicks on a specific weapon the Buy/Equip Popup window is displayed and the Header, Left Selection and Main Display panels are locked.

In Equip Mode the Left Selection Panel displays the weapon slots and the Main Display panel displays owned weapons which are allowed in the selected slot. A green check mark is used to show the weapon equipped for that slot (if any).

Store Assets

The primary assets for the store can be found in the folder DB Upgrades_Store->Store and are described below.

  • DBStoreController.js - This script contains the non-GUI components of the store and and serves as the primary interface between the GUI and the rest of the weapons package. The functions in DBStoreController can also be used to leverage the store functionality within any game specific scripts. The key functions are listed below.
  • activateStore()
  • deActivateStore()
  • buyWeapon()
  • sellWeapon()
  • buyUpgrade()
  • equipWeapon()
  • getBalance()
  • deposit()
  • withdraw()
  • DBStoreGUI.js - This script manages the user input and draws the GUI. 
  • SlotInfo.js - This script is used to control which Weapon Classes are allowed in which slots. This script has an associated custom editor in the Editor folder called SlotInfoEditor.js.
  • WeaponClassIcons.js - This script is used to manage the optional icons representing the user configurable Weapon Classes (Shotgun, Handgun, etc).
  • StoreController - This prefab has the above scripts attached and configured with initial values. To add the store to a scene simply drag in the prefab.
  • Images - This folder contains some sample images and textures used for the demo store.

Configuring the Store Layout 

Although the store will work with no additional configuration the package is designed to make it easy to customize the look-and-feel for your game. To make changes simply select the StoreController prefab and adjust the values in the DBStoreGUI component. 

The following parameters can be adjusted without modifying any code.

  1. Skin- GUISkin used for the Store. The demo scene uses the default skin but a custom skin could be selected to change all aspects of the store.
  2. Checked Icon - The Texture displayed if a weapon is owned or equipped. A green check is provided by default.
  3. Locked Icon - The Texture displayed when a weapon is locked. A sample Texture is provided.
  4. SPos X - the upper-left corner X position of the store. If either SPos X or SPos Y are negative the store is centered.
  5. SPos Y - The upper-left corner Y position of the store. 
  6. HWidth - The width in pixels of the Header Panel.
  7. HHeight - The height in pixels of the Header Panel
  8. HTitle Height - The height if the Header Title
  9. HTitle - Text for the Header Panel Title.
  10. HTitle Image - An optional Image for the Header Panel Title. If not defined the HTitle text is used.
  11. LSHeight - The height in pixels of the Left Selection Panel (LS)
  12. LSWidth - The width in pixels of the LS
  13. LSButton Height - The height in pixels of the buttons in the LS 
  14. LSButton Width - The width of the buttons in the LS
  15. LSxPad - X padding between elements in the LS
  16. LSyPad - Y padding between elements in the LS
  17. LSNextIcon - Icon for next page in LS if needed
  18. LSPrevIcon - Icon for previous page in LS if needed
  19. MDButton Width - Width of Buttons in the Main Display Panel (MD)
  20. MDButton Height - Height of Buttons in the MD
  21. Popup Upgrade Width - Width of Buy/Upgrade popup window
  22. Popup Upgrade Height  - Height of Buy/Upgrade popup window
  23. Upgrade Buy Icon - Optional icon used for the Buy Upgrade button in the popup window
  24. Upgrade Info Icon - Optional Icon used for the View Upgrade button in the popup window

Configuring Weapon Categories, Weapons and Upgrades for the Store

This section describes how to configure the weapons and upgrades to display properly in the store.

1) Add the StoreController Prefab to your Scene

Drag the StoreController prefab from the DB Upgrades_Store->Store folder into the top level of the scene.

2) Define the Weapon Categories for your Game

Define the Weapon Categories for your game by editing weaponInfo.js in the folder ScriptsDB->WeaponScripts. The categories are defined in an Enumeration weaponClasses

// You can change the WeaponClasses enum to define your own // Weapon Classes,

// "Null" must be the last value in the enum and should be

// applied to one empty weapon object.

// The store will replace any underscores with a space in

// the display name. Sniper_Rifle will be displayed as //"Sniper Rifle"

enum weaponClasses {Shotgun, Handgun, Assault_Rifle, Marksman_Rifle, Explosive, Melee, Special, Null}

3) Optionally Supply Images for the Weapon Categories.

Select the StoreController prefab in the scene and then drag an image into the Texture slot for Weapon Class

If these textures are defined they will be displayed in the Left Selection Panel of the Store. If they are missing the text names will be used.

4) Configure the Number of Weapon Slots

The number of Weapon Slots is defined in the playerweapons.js script on the Weapon Camera. 

If you already have the slots set up for your game, then don't worry; the Store will automatically pick up the number of slots from playerWeapons.

To set the number of slots simply select playerweapons on the WeaponCamera in the scene and change the Size of the Weapons array. By default the player can select the weapon in a slot using the number keys ( 1 for slot 0, 2 for slot 1, etc). 

5) Configure the Weapon Classes Allowed in Each Slot

The Store restricts the weapons that can be equipped in a slot based on it's weapon class. The restriction is based on the configuration of the SlotInfo component on the StoreController prefab. 

To configure simply select the Store Controller in the scene and then fill in the Slot Name and allowed Weapon Classes for each slot. 

If the Slot Name isn't defined it will default to "Slot N".  

The Allowed Weapon Classes defaults to none selected so make sure you select at least one class for each slot!

6) Configure the the weaponInfo Values for each Weapon in the Scene

The weapons available in a scene are configured under the Weapon Camera attached to the Player. 

For each weapon you must configure a number of fields on the Weapon Info component. If the values have already been set the store will pick them up automatically.

  1. Owned - This field is set to true by the store when a weapon is purchased.
  2. Locked - If 'locked' is selected a locked icon is displayed on the weapon 
  3. Weapon Class - Select a value from the drop-down (These are the values you defined in step 2 above). **Note that you must have one Weapon named Null with the Weapon Class Null selected.
  4. Gun Description - This description is displayed in the store.
  5. Locked Description - This description is displayed when a player tries to buy a locked weapon. 
  6. Gun Name - Self explanatory
  7. Buy Price - The cost to purchase the weapon with no upgrades applied
  8. Sell Price - The sell price of the weapon with no upgrades applied
  9. Sell Price Upgraded - The total sell price of the base weapon and all owned upgrades.
  10. Icon - This is an optional image for the weapon. If an image is defined it will be displayed in the store, if not, the name of the weapon is used. An example Icon is supplied in the Store->Images folder.

7) Check the Values on the Upgrades for Each Weapon

The Store will automatically find all weapons and all upgrades on each weapon. Instructions for defining and configuring upgrades is provided elsewhere in this document, however, this section describes the configuration as it applies to the store. **Note: AimObject is on all weapons and is not an upgrade.

  1. Applied -This field is set to true by the store when an upgrade has been applied to the weapon
  2. Owned - This field is set to true by the store when the upgrade is owned.
  3. Locked - If this field is set to true the upgrade will be displayed in the store but it will not be purchasable. In your game you could lock an upgrade until the player has achieved a certain goal and then set locked to false.
  4. Description - A description of what the upgrade does. Since the stats of the upgrade aren't displayed (the Store doesn't know which stats are worth displaying) be sure that the description fully describes what the upgrade does.
  5. Locked Description - A description of why the upgrade is locked. This could be used to tell the player what they need to do to unlock the upgrade.
  6. Buy Price - The price to buy the upgrade.
  7. Sell Price - The sell price of the upgrade. Note: The store doesn't allow users to sell an upgrade, the must sell the weapon and all upgrades together. 

Configuring Weapon Pickup

Weapons can be placed in a scene for players to pickup. Once the weapons are picked up they are deleted from the scene and treated in exactly the same way as a weapon that was purchased in the store. They can be upgraded, equipped and un-equipped or even sold.

This section explains how to configure a weapon for pickup. 

Weapon Pickup prefabs are provided in the PrefabsDB-Weapon Pickups folder. Each weapon Pickup consists of a gun model, a "Highlight" copy of the gun model, a box collider and the SelectableWeapon.js script.

The Highlight model is a copy of the weapon model that has the textures replaces with a lighter texture and is displayed when the weapon is selected for pickup.

To add a pickup weapon to the scene follow these steps.

  1. Place a Weapon Pickup prefab in the scene
  2. Drag an actual weapon into the 'weapon slot' of the SelectableWeapon Script in the inspector.
  3. Make sure the prefab is tagged "pickupAble"
  4. Make sure the Weapon Camera has the PickupWeaponStore.js script attached.
